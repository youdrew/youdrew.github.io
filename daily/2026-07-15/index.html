Claude Code v2.1.210 发布

Claude Code 发布 v2.1.210 版本，重点针对隔离模式与权限系统中的安全缺陷进行修复，并改进长时间工具调用的可见性。在安全方面，此前 worktree 子代理可绕过沙箱对主仓库直接执行 git 写操作，破坏隔离假设；此次更新强制子代理仅能修改自身 worktree，降低跨工作区污染风险。同时，ultracode 关键词此前可被 webhook 负载、转发 PR 评论等非人类来源触发，存在被外部输入滥用之嫌，现已限定为人类显式启用。权限规则层面，写入类工具的 path 通配符（如 Write、NotebookEdit、Glob）被改为推荐使用 Edit 或 Read 的等效规则，启动时会输出告警，避免因权限粒度不当导致意外覆盖。产品体验上，折叠态工具摘要行新增实时计时器，长时运行的命令不再像卡住一样停留在原状态，提升调试可观测性。此外还修复了渲染文本片段泄露到 crash 报告的边缘问题。整体看，本次更新延续"安全优先、可观测性次之"的迭代路线，体现出对沙箱边界与权限语义的持续收紧。

09Apple 机器学习

苹果提出Pare框架：模拟主动用户评估主动型智能体

苹果机器学习团队发布论文，提出Proactive Agent Research Environment（Pare）框架，旨在为能主动预判用户需求并自主执行任务的智能体提供研发与测试环境。研究指出，现有的智能体评估方法普遍将移动应用抽象为扁平的"工具调用"接口，仅暴露零散的函数入口，既无法反映应用界面与状态之间的依赖关系，也难以模拟用户点击、滑动等具有时序性和上下文性的真实操作，导致针对主动型智能体的用户模拟在实践中几乎不可行。Pare将每个应用建模为有限状态机，明确描述界面状态、可用操作及状态转移规则，使智能体能够在接近真实使用逻辑的环境中预测用户意图并提前介入；在主动智能体有望率先落地的数字助手场景下，这类框架的提出意味着行业开始正视从"被动响应"到"主动代办"的能力跃迁所依赖的评测基础设施缺口。

Apple Music多语言语义检索系统

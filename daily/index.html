Stripe 的 AI 一把手 Emily 上了 MAD Podcast 跟 Matt Turck 聊 agentic commerce——简单说就是 AI 不光帮你找东西,还要替你去跟对面的 AI 谈合同、付钱、收货。她给出了一个类似自动驾驶 L1 到 L5 的分级框架,消费侧现在主要停在 L2(AI 帮你挑、你点确定),L3 已经开始落地,比如 ChatGPT 的 Instant Checkout 和 Gemini 里的 Buy button,而企业那边的 L3 已经规模化,因为「买货 + 合同 + 合规」天然适合 agent 来跑。她还提到 Stripe 跟 OpenAI 一起搞的 Agentic Commerce Protocol(ACP)和 Shared Payment Token,商家一次接入、所有合规的 agent 都能用,token 里直接编码好「能在哪家店、花什么币、多少钱、用多久」,商家拿到 token 却看不到你真实的支付凭证。Stripe 的 Link 全球有 3 亿用户,现在升级成了「agent 钱包」,能按商户、按地理位置、按金额上限来设规则,每笔交易还带 Radar 的风险评分——Emily 抛出一个扎心数据:AI 公司的新注册里有超过六分之一是来薅羊毛的(偷 token、批量刷免费试用),过去半年免费试用滥用直接翻倍,她把 token theft 称作「当下 AI 最被低估的话题」。

Emily 还把 vibe coding 和 vibe deployment 拆开聊,说写代码这关其实已经过了——Stripe 文档站的 agent 流量一年涨了 10 倍,现在占总流量 40%,CLI 里 70% 的请求都是 agent 发来的,真正的瓶颈变成了「部署」,所以他们做了 Stripe Projects,让 agent 从命令行就能把 Supabase、Vercel、Cloudflare、Twilio、Clerk 这一整套服务编排起来,绕开那些「为人手设计」的后台。她顺带吐槽了 SaaS 按席位收费的老模式:每次推理都有边际成本,Lovable、ElevenLabs 这些公司已经开始走「订阅 + 按用量计费」的混合模式,等消费方变成 agent 而不是人,月底结算根本来不及,必须实时计量加实时结算(Stripe 还跟 Tempo 链上稳定币合作)。最后她放了个大背景:美国新增企业 100% 来自一人公司,现在有 500 万人靠 solo business 养家,数十万人年收入过百万美元,AI 的真正贡献不是消费侧那点增量,而是商业组织形式的根本变化。整期听完你会发现,Stripe 这盘棋不是「再做一个新服务」,是把「提升互联网 GDP」这个老使命延伸到了 agent 时代的新瓶颈上,野心是真的大。

Dwarkesh Patel 这期请来了 Sarah Paine,聊一个挺反直觉的地缘话题:为什么陆权国家打仗往往亏到破产,海权国家却常常闷声发财。她从历史案例一路拆到后勤、成本和战略纵深,听完你会对「为什么打仗这么贵」这件事多一层理解。

Matthew Berman 直接让 GPT-5.6 搓了一整个 Minecraft,从地形、生物、合成表到基本玩法全是 AI 一把生成,跑起来居然还能玩。属于那种看完会沉默三秒、然后感叹「时代真的变了」的演示。

🎙️ The MAD Podcast with Matt Turck:Stripe's AI Chief — How AI Agents Will Buy, Sell, and Pay · 嘉宾 Matt Turck

**一句话要点:**Agentic Commerce 正在从「AI 帮你找商品」快速过渡到「AI 替你和另一边的 AI 谈合同、付钱、收货」,而 Stripe 已经把整条支付与身份基础设施铺到了门口。

**为什么值得听:**Emily 是 Stripe 数据与 AI 一把手,过去一年 Stripe 几乎所有 agentic commerce 的公开动作——Agentic Commerce Suite、Shared Payment Token、Link Wallet for Agents、Stripe Projects(也就是「vibe deployment」)、还有跟 OpenAI 共同设计的 Agentic Commerce Protocol(ACP)——背后都有她。她既是这套叙事的对外发言人,也是 Stripe 内部把这些事情真正工程化的人。听众能从她这里同时拿到「经济学视角的合理性论证」和「工程基础设施该怎么搭」。

**Protocol 层:ACP 和 Shared Payment Token。**Stripe 与 OpenAI 去年共同发布了 Agentic Commerce Protocol(ACP),一个与平台、支付通道都解耦的开放标准——商家一次接入、所有合规 agent 都能用。ACP 解决两件事:一是商家如何确定性地暴露目录、库存、价格(而不是让 agent 靠推断);二是 Shared Payment Token,一种新的支付原语,把「agent 可以对哪个商家、用什么货币、花多少、用多久」这些规则编码进 token 本身,商家在 checkout 时拿到 token 但永远看不到底层凭证。Emily 反复强调,ACP 是 platform-agnostic、PSP-agnostic 的——你可以把 token 转给 Adyen 处理,不必走 Stripe。

**Vibe Deployment:Stripe Projects。**Emily 把 vibe coding 和 vibe deployment 区分开:写代码这一步其实已经被解决了(Stripe 文档站的 agent 流量比一年前涨了 10 倍,现在占总流量 40%;Stripe CLI 的资源请求里 70% 来自 agent),真正的瓶颈变成「部署」。Stripe Projects 是个编排层:agent 从命令行就能注册并配置 Supabase、Vercel、Cloudflare、Twilio、Clerk 这一整圈服务,绕开那些「为人手设计」的开后台流程。Stripe 自己的解释是:这条产品线的战略意义不是「做一个新的 Stripe 服务」,而是「把 Stripe 的使命——increase the GDP of the internet——延伸到一个新的瓶颈环节」。

**经济学终局:商业活力(business dynamism)。**Emily 引用 Census Bureau 的数据:美国新增企业增量完全来自 non-employer firms(也就是 solopreneur),现在美国有 500 万人靠一人公司养家,其中数十万人年收入过百万美元。AI 的双向作用是:vibe coding/vibe deployment 降低「从想法到产品」的门槛,domain-specific agent(会计、客服、运营)降低「从产品到生意」的运营成本。她认为这才是 AI 对 GDP 的真正贡献——不是消费侧的增量,而是商业组织形式的变化。

嘿,今天这期 AI Builders Digest 的料也太猛了,从命令行工具逆袭白板,到 NVIDIA 把开源模型当成”反向猜芯片需求”的工具,每条都想跟你唠两句。话不多说,咱们一条一条来,顺便认认这些在圈子里很有发言权的人。

先从 Swyx 开始,他是 Latent Space 播客和 AI Engineer 社区的标志性人物。这条吐槽特别有意思:搞 “tools for thought” 那拨人折腾了十年漂亮 canvas(可视化白板型工具)demo,最后被那些长得丑、对比度拉胯的 CLI(命令行工具,就是让你在黑窗口里敲命令的那种工具)吊打,为啥?因为后者能直接替用户把那些 commodity thinking(基础思考活)给做了。后续两条他还发了个跟 Thariq、Simon Willison、Cat Wu 合作的文章链接,顺便感叹了一句 Factorio(《异星工厂》,一款工程建造类游戏)怎么就”破圈”了。明明是想提升效率的工具圈,最后赢的是最朴素的那一拨,挺反讽也挺真实。

接下来是 Linear 负责人 Nan Yu,他连开三炮:第一炮吐槽 Tailwind 被搞出 Bootstrap 的那帮人收编,堪称 Tailwind 的”最高胜利”;第二炮直接下结论,”如果一个领域产不出好的训练数据,那说明这领域本来就在装神弄鬼”;第三炮聊 LLM 时代的理想医生,想象一下——一个愿意花大量时间钻研你病历的医生,再配上 LLM 当工具。不得不说,他这几条嘴是真毒,金句密度也高。

转场到 Anthropic 这边的 Cat Wu,她是 Claude Code + Cowork 的负责人。她先抛了一个实用小贴士:用 Claude Code 加上 computer use(让 AI 直接操控你的电脑界面)就能配好 Claude Tag,把文档甩给它,它自动帮你把团队的 GitHub repo、数据仓库、Google Drive 这些数据源接好,挺省事。她还顺手开了个 Fable 5 长周末 demo 征集帖,喊所有 builder 回复里晒作品,氛围挺好。

那接 Fable 5 这个话题,Anthropic Claude Code 团队的 Thariq 紧接着发了一篇长文,他是 YC W20 的校友。他讲的是自己用 Fable 时最关键的技能——先发现自己”不知道什么”,才能 prompt(提示)得更准。他甚至用 HTML artifact(Claude 生成的可交互小网页)来帮助自己把没想清楚的问题摆到台面上,然后把这套打法串回了自己在 AI Engineer(AIE)大会上的 talk,还特意感谢了 Geoffrey Litt 给的反馈,促成了文章里的 quiz 环节。这条挺适合正在跟 AI agent 磨合的人看。

再来看 Replit 这边。CEO Amjad Masad 一句话公告:Replit 上线视频生成功能,后面跟了个试用链接,真的是一句话,不废话,CEO 的战斗力都在产品里了。

Vercel CEO Guillermo Rauch 这次在推一个新概念叫 agentic self-improvement,直译就是”agent 自我进化”:让 agent 能回头看自己的运行历史,找低效、错误、冗余的 tool call(工具调用,agent 调用外部 API 或软件的那个动作),然后自动产出新的 prompt 和 skill(技能)。他把这事直接挂到了 Vercel 部署里新出的 agent 可观测性(observability,帮你看清 agent 每一步在干啥)上。另外两条跟 AI 无关——一条是 7 月 4 日对《独立宣言》的感慨,一条是把 JSX 那种把 HTML 直接写进 JS 里的写法类比成”理想航空体验”的吐槽,挺有 Rauch 的味道。

下面这位是 Box 的 CEO Aaron Levie,他那条标志性的长 thread 这次主题特别正:”AI 的终局之争是 context(上下文)之战”。核心论点是这样的:agent 越来越强,真正能赢的是那些能把领域上下文捕获好、治理好、并持续优化的平台,不管你做的是 coding agent、legal agent 还是 support agent。然后他预言了应用层的走向:大模型路由该这么打——规划和调度用前沿大模型,日常执行切去更便宜的模型,开源闭源都行,而且会越来越多地自己训领域专用模型。结尾还给 FDE(Forward-Deployed Engineer,驻场工程师,专门帮你把 AI 真正落到业务流程里的人)打了个 call,认为这才是应用层的真护城河。

Garry Tan,Y Combinator 总裁兼 CEO,这次的关注点在医疗。本批数据最好的那条他说的是:专科医生等待时间正在变长,而 AI 偏偏在这个节骨眼上要把这事改写,他期待 AI 把医疗质量拉升 100 倍,虽然有点夸张,但”对所有病人来说这时间点都不算早”这句挺打动人。另外两条偏政治,一条是 NIMBY(Not In My Back Yard,”别建在我家后院”,反对社区附近搞开发的那种人)住房批评,一句是”defund harm acceleration”的口号,跟 builder 关系不太大。

接下来是 Zara Zhang,builder,Harvard ‘17,而且她就是这套 Follow Builders skill 的作者。她抛了两条挺尖锐的观察:第一条,现在用户越来越不愿意为”工具”买单,因为 coding agent 能自己搓一个,他们真正愿意买的,是”雇到一个自己没有的专家”那种感觉;第二条,每天发推一旦过了最开始的坎,就不再是任务,而像戴上了一副 lens(滤镜),她一天平均发三条,idea 自然就冒出来,挺有意思的产品视角。

Nikunj Kothari,FPV Ventures 合伙人,前 Meter / Opendoor / Atlassian 早期员工,他有两条值得记的观点。第一条有点”凡尔赛”地表扬 Google:虽然他是 Gemini 产品体验的公开批评者,但 Gemini 仍然是唯一一家”一张 API key 走天下”的——Flash 跑快速结构化任务、Nano Banana(Google 图像生成模型)做世界级生图、带 grounding(搜索增强,模型回答前先搜一手信息再答)的 Search、再加上 realtime 音视频,全齐了。第二条是个半开玩笑的吐槽:大模型厂似乎专门挑长假前一天发新模型,Opus 4.5 卡在感恩节前,Fable 5 卡在 7 月 4 日前,正好给群众留时间折腾、玩坏、然后集体陷入 token 账单焦虑。

Peter Steinberger,资深 iOS 开发者,现在 OpenClaw 和 OpenAI 两边跑,他在猛测 Fable。本批最出圈的是这条:”如果你觉得 Codex 设计丑,让它’用 imagegen 重新想象这个设计然后实现’试试。”这是 Codex CLI(OpenAI 的代码生成命令行工具)圈最近流行的小套路,Codex 最弱的就是设计维度,所以直接甩给 image generation 模型补救。他还把自己 8 万条推都喂给 Fable,让它更狠地 roast(吐槽)自己,挺豁得出去。

Every 的 Dan Shipper 现在是 Fable 5 发布周的状态。他在辟谣一个流传的说法:”Fable 根本不是同一个模型”——其实是同一个,只是回退到 Opus 4.8 的频率比之前高了点,所以 benchmark 数据是混着两个模型的,他吐槽说这其实是个 skill issue(操作者水平问题)。他还甩了一份 Fable 5 prompt 库让大家带去海边,再配一条按 token 计费的”信用卡广告体”爆款帖:”全新 iOS app 端到端跑通 5M tokens、清掉整条生产 bug 积压 20M tokens、对每封未读邮件都写超详细回复 30M tokens、Fable 在泳池边替你干活无价。”这文案是真的会写。

Claude 官方账号这期在推 Squidsoup——一个做沉浸式声光空间体验的艺术组合,在伦敦南岸中心跟一支管弦乐队合作做了大尺度现场,挺值得刷一遍,看看”Claude + 艺术家”在真正生产场景里是什么样。

最后压轴播客这条来得正合适——MAD Podcast with Matt Turck 采访了 Bryan Catanzaro,NVIDIA Nemotron 的负责人。Nemotron 3 Ultra 几周内就成了美国最强的开源权重模型(Nemotron 是 NVIDIA 自家的开源基础模型家族)。聊的核心是:NVIDIA 做模型不是为了抢市场,是为了把自己对 AI 的理解搞透,从而反向推导下一代的芯片、系统、编译器该怎么设计,毕竟每多一个 AI 部署场景,未来就可能多一份 Blackwell(Blackwell 是 NVIDIA 当前一代旗舰 GPU 架构)部署。他对”中国就是蒸馏我们”这种说法很不以为然,自己早年跟 Andrew Ng、还有当时还年轻的 Dario Amodei 在百度硅谷 AI 实验室一起共事过,他认为中国开源社区是真在推动整个领域往前走。

技术侧 Nemotron 3 这一代堆得挺满:NVFP4 的 4-bit 预训练(NVIDIA 自研的 4 位浮点格式,训练时把数字精度直接压到 4 位,显存和能耗都省一截);Mamba 加 Transformer 的混合架构(以 state-space 模型为主、配少量 full attention,组合起来比单独任何一种都更”聪明”);MoE(Mixture of Experts,混合专家模型,把模型切成很多”专家”子网络,每次推理只激活一小撮省算力)再加 “latent MoE” 创新(把 token 在 NVLink 上的传输先压缩再走,同样推理成本能塞下 4 倍专家数);100 万 token 上下文窗口;多 token 预测(Multi-Token Prediction:模型一次性预测多个 token,用下一轮验证,能拿到大约 4 倍加速,而且模型越准速度越快——这是只有加速计算思维下才会出现的奇怪特性);以及跨 10 到 15 个领域专家用 MOPD(Multi-teacher On-Policy Distillation,多师蒸馏)做的多师蒸馏后训练。

最有意思的是他把多师蒸馏也当成了一种组织管理工具来看——它让 500 多个研究员在科学、数学、coding、agent harness 各条线并行干活,避免出现”这轮训练到底优先谁的工作”这种扯皮。他还直言不讳地说:MoE 现在已经是事实上的前沿架构,虽然在显存和小 batch 下都有短板,但这条路已经回不去了。

好啦,今天的博主和播客动态就是这些,从 Swyx 那句”丑 CLI 逆袭白板”的凡尔赛,到 Catanzaro 把蒸馏当组织工具的硬核观点,信息密度拉满。下期咱们接着拆,毕竟 AI 圈这帮人,今天发吐槽明天就能给你写个新产品,根本停不下来。

“AI 这场仗，归根结底是 context 的仗。所有事情最终都回到一个点：怎么让 agent 尽可能有效。”

The Takeaway: NVIDIA is in the model business not to dominate but to understand — building open models like Nemotron is how the company reverse-engineers the future of accelerated computing, and “speed-first” is now a hard architectural constraint, not a marketing line.

Bryan Catanzaro leads Nemotron, NVIDIA’s family of open foundation models — Nemotron 3 Ultra became the #1 US open-weights model within weeks of release. His argument for why NVIDIA ships open models at all is sharp: NVIDIA’s first job is to deeply understand AI so it can keep designing the chips, systems, and compilers that make AI go. The second job is to keep the open ecosystem alive, because every AI deployment is potentially a Blackwell deployment. He pushes back hard on the “China just distills us” narrative — having worked at Baidu’s Silicon Valley AI Lab alongside Andrew Ng and a young Dario Amodei — and credits the Chinese open community with genuinely pushing the field forward.

On the technical side, the Nemotron 3 stack is dense: 4-bit pretraining in NVFP4（NVIDIA 的 4 位浮点格式，模型训练时把数字精度压到 4 位，大幅省显存省电）, a hybrid Mamba-transformer architecture (mostly state-space, with a dash of full attention, because the combination is smarter than either alone), MoE with a “latent MoE” trick that compresses token traffic across NVLink so you can fit 4x more experts at the same inference cost, a 1M-token context window, multi-token prediction that gets you a free ~4x speedup when the model is accurate (the more accurate the model, the faster it runs — a uniquely “accelerated computing” property), and multi-teacher on-policy distillation across ~10–15 domain specialists using MOPD.

The most refreshing part is the org-design lens. Catanzaro frames multi-teacher distillation not just as a technique but as a coordination mechanism — it lets 500+ researchers across science, math, coding, and agent harnesses work in parallel without a tug-of-war over which team “wins” each cycle. He’s also candid that Mixture-of-Experts is now the de facto frontier architecture, despite its memory and mid-batch-size tradeoffs.

“If you accept as the truth that we’re gonna be running at the limit, then what that means is that the way to get more intelligence is to be more efficient. We can’t get more intelligence by applying more force if we’re already at the limit.”

第二条和第七条我觉得可以并到一起聊。AgenticSTS 这篇是给”长任务大模型智能体”搭了一个有界记忆的测试床,它提醒我们一件事:智能体的记忆本质上是一个”未来每个决策能看到什么”的契约,简单粗暴把历史观测、工具调用、反思全塞进 prompt,信息倒是全了,但很容易变成一锅粥,模型根本抓不住重点。SkillCoach 关注的是另一面,也就是智能体怎么”用技能”:现实里技能库动辄上百条,彼此还重叠,光看最后任务成没成,既不好评估也不好训练,所以它让智能体自己进化出一套评分细则来打分。这两条加起来其实在说同一件事:智能体能不能做长任务,关键不在模型够不够大,而在它会不会管理自己的上下文和技能,我觉得这是今年最被低估的研究方向之一。

第五条 Multi-Resolution Flow Matching 是给文生图扩散模型做加速的,而且特别强调”不需要重新训练”。它走的是多分辨率采样,先生成低分辨率大结构,再补细节,这样就能跳过很多计算步骤,还不用动硬件不用改系统,算是那种工程上很友好、立刻能用的优化,我对这种”白嫖性能”的工作天然有好感。

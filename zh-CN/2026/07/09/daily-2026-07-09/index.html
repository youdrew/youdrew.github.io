Meta 推出 Muse Image 和 Muse Video,这是他们超级智能实验室发布的首个媒体生成模型。Muse Image 号称目前最先进的图像生成,能精准听话、按多张参考图重排画面,还能读懂 Instagram 上的社交上下文来帮你出图;Muse Video 在同一个底座上加了原生音频支持,视觉也很能打。视频和图片都已经可以在 Meta AI 应用、网页、Instagram 动态和 WhatsApp 里试用了,不过先在部分国家上线。这波动静不小,Meta 终于把自家生成模型端到端摆出来了,就看实测能不能打得过 GPT 和 Gemini 那一档。

接下来要说的是 MIRA,一个可以真的玩起来的多人世界模型,被形容成”火箭联盟的梦”。它靠一万小时机器人公开数据训练,能实时根据你的按键渲染四人对战的画面,帧率跑到 20 FPS,由 General Intuition 和 Kyutai Labs 一起搞的,Epic Games 也参与了协作。从最早的扩散版 DOOM 玩到现在的多人实时,这条线真的越来越有意思了,演示、技术报告和代码全开源,ICML 现场也能去玩。

Rowboat 是一个开源、本地优先的桌面 AI 助手,简单说就是装在你电脑上的 AI 秘书。它会把邮件、会议、Slack 这些数据整理成像 Obsidian 那种笔记图谱的样子,给你一个长期记得住上下文的记忆;里面还自带邮件客户端、浏览器、会议记录器、连 Claude Code 和 Codex 都能调用,后台还能按事件或定时跑代理任务,想接 Exa 搜索、GitHub 这些外部工具通过 MCP 协议就能挂上。所有东西都以纯 Markdown 存在本地,不会绑死服务商,模型可以用本地 Ollama、LM Studio,也能挂 API 密钥用云端模型。本地派可以冲一波。

Grok Imagine 更新了,Grok App 记得升一下,现在可以做 15 秒的视频了,马斯克直接说质量难以置信。视频生成赛道又卷出新高秒数,值得点进去看看。

Claude Cowork 开始向移动端和网页端开放了,会话和文件能跨设备同步,先给 Max 计划的用户开 Beta。它能干的事情是让 Claude 跨文件、日历、邮件、即时通讯这些工具跑复杂的活儿,有意思的是超过九成的使用场景其实不是写代码,而是日常知识工作,比如业务运营和内容创作。在电脑上开任务,手机看进度,合上笔记本它还能接着跑,周一早上六点自动帮你把客户简报准备好;遇到要你拍板的事,会直接发到手机上提醒你。这波就是冲着把 AI 同事装进口袋去的,网页端也把聊天和 Cowork 合并成统一界面了。

Google 给 Gemini API 的 Managed Agents 加了一组新能力。后台执行让任务异步跑,传一个 background true 就立刻拿到任务 ID,可以轮询或流式看进度;还能直接挂远程 MCP 服务器,不用自己写中间件,跟 Google 搜索、代码执行这些内置沙箱工具随便混着用;自定义函数调用支持本地执行业务逻辑,内置工具在服务端跑;凭证刷新也做了,环境 ID 一传、网络配置更新一下,沙箱里的文件、装好的包和仓库都还在。这一套基本就是奔着生产级 AI 智能体去的,开发者可以省不少事。

NotebookLM 的短视频概览功能正式上线,移动端和网页端的英语用户都能用了,之前一直在灰度,现在全量铺开,大家可以去玩,顺便给官方留言想要哪些新功能。

Hugging Face Storage 现在成了 SkyPilot 的一级后端,简单理解就是 SkyPilot 帮你把训练和推理任务调度到二十多个云、Kubernetes、Slurm 还有本地集群的任意 GPU 上,数据可以直接从 Hugging Face 的 Bucket、模型、数据集仓库挂进来,用 hf 加冒号加路径这种 URL 就行,支持按需懒加载或者一次性拷贝。而且 Hugging Face 不收出站费和 CDN 费,跨云读数据基本没额外成本,存储价格一个月 12 到 18 美元每 TB,比 AWS S3 加出站费还便宜,对多云训练的人来说是真省钱。

Claude Code 出了 v2.1.203 版本。修了几个让人抓狂的 bug,比如 macOS 下因为内存检测误报让后台会话卡个十几二十秒、后台会话因为 token 过期直接永久报废现在能自动恢复了、交互式会话里那个上下文指示器每轮把整段对话重新分析一遍吃 CPU 内存也都修了。另外左侧箭头键以前一按就把后台任务、差异和工作流详情直接关了,现在改成按 Esc 才关,顺手还把启动体积压了大约 7 MB、内存也少了 7 MB,日常体验会顺滑不少。

中国这边打算限制外国人访问他们最强的 AI 模型。商务部牵头,跟阿里巴巴、字节、Z.ai 这些公司都谈过了,先进模型包括还没发布的都想留在中国国内,而且不只是限 API,连可下载的权重都想管住,泄漏甚至可能按危害国家安全论处;同时还在讨论限制外国资本投资中国 AI 初创。华盛顿那边本来就在限制美国先进模型出口,这一来全球 AI 市场可能要再撕开一条缝,后面怎么走值得盯着。

Ethan Mollick 发了一条很短的提醒,大意是不要指望前沿开放权重模型能一直这么自由地流动下去,这句话跟前面那条中国管制新闻放一起看就更值得琢磨了。

微软正在用自家 MAI 模型替换 Copilot 产品里的 OpenAI 和 Anthropic 模型,目标很直白,就是为了省钱。MAI 已经在 Excel 和 Outlook 里每周跑好几万次请求,虽然占比还小;Build 大会上发布的推理模型 MAI-Thinking 1 号称编码能跟 Sonnet 4.6、Opus 4.6 打,但基准测试其实落后不少,差不多就是 DeepSeek V3.2 的水平。微软 AI 负责人也承认这就是要削减并最终干掉对 Anthropic 的支出,未来有可能默认跑 MAI,第三方模型要付费才用,这盘账算得相当精。

美国第一批自主地面车辆已经在乌克兰战场上用了。自动驾驶公司 Forterra 把一百多辆基于 Polaris ATV 改的 Lancer 自主车送过去,能拉 750 公斤货,加装 Starlink 天线远程操控,九个月下来跑了两千五百英里,送了近 78 万磅物资,还做了 52 次伤员撤离。目前主要还是远程操控为主,自主系统还没法实时识别和应对敌方威胁,但已经在练了,公司融了五个多亿美元,正把传统机器人方法和生成式 AI 结合起来。地面自主技术算是真上过战场了,这件事比新闻标题要重得多。

Elvis Saravia 介绍了一个用 human-in-the-loop 提升 agent 循环可靠性的玩法,具体就是把他所有的 Claude 和 Codex agent 会话都挂到 DialAgent 这个 MCP 服务器上,服务器会给 agent 分配一个真实电话号码,能打电话、发短信、走 iMessage。当 agent 在循环里处理 PR 或者新功能拿不准的时候,会直接打电话升级到人,人不在电脑前也能接招。这个思路挺巧的,等于给 agent 装了个真人热键,五美元免费额度就能去试。

AI 公司在 12 个月里合计承诺 97.5 亿美元建设前部署工程团队,也就是 FDE。现在出现了三种结构,微软、亚马逊这种从现有编制里调人,还有 Salesforce 承诺一千个 FDE 岗位算资产负债表型;OpenAI 融了 40 亿美元做 Deployment Company 投后估值 140 亿,Anthropic 从黑石那边融了 15 亿美元,算独立实体型;Google Cloud 直接拿出 7.5 亿美元做合作伙伴基金。瓶颈已经从模型能力转移到部署,模型已经够强,难的是企业自己装不上调不起来,这也是未来一年 AI 公司真正拼护城河的地方。

YC CEO Garry Tan 在 X 上说自己跟 AI 编码代理每天在五个项目里部署 37000 行代码,还保持了连续 72 天发版的纪录。结果波兰开发者 Gregorein 把他网站前端扒开一看,169 次请求、6.42 MB 数据,对比 Hacker News 才 7 次 12 KB;还有 28 个测试文件、78 个压根没用的 JavaScript 控制器、八种格式的 Logo,未压缩的旧 PNG 也堆在里面。AI 写代码快是快,但质量真的不能被数量盖过去,这事儿挺值得所有用 AI 编程的人对照一下。

Liquid AI 开源了一个叫 Antidoom 的小工具,专门修推理模型里的死循环 bug,核心是一种叫 Final Token Preference Optimization 的方法,定位循环开始的第一个 token 训练模型换条路走,不破坏整体输出分布。在他们自家的 LFM2.5-2.6B 模型上,硬数学和编程的循环率从 10.2% 砍到 1.4%,Qwen3.5-4B 上从 22.9% 砍到 1%,整套流程几个小时就跑完,代码和数据都开源了。做推理模型应用的可以重点看一下,死循环是真的烦人。

在不稳定的网络地区,小型 AI 模型正在悄悄普及。2019 年尼日利亚人 Adebayo Alonge 因为服务器在欧美让验药应用 RxScanner 一次扫描要五分钟,工程师两小时把模型压到能在安卓手机上跑,从此无网无电也能验药。小模型参数一般几十亿以内,手机和树莓派就能带得动,功耗几瓦;印度有用它做腰果病害无人机的、乌拉圭识别入侵蚂蚁的、疟蚊检测、巴西用 Arduino 做的心电图设备。世界银行的数据说最穷国家只有 0.7% 的网民用过 ChatGPT,发达国家是四分之一,小模型才是把 AI 救命服务送进缺算力缺电地方的真正答案。

Krea 2 的身份保留功能上线了,模型和 ComfyUI 节点也一起发,做一致性角色的朋友可以试试。

Claude 官方发了一份基于 5 月 11 日到 31 日 120 万次匿名会话的分析,看大家到底怎么用 Cowork。结果最大用途是业务流程与运营,占 33.4%,就是整理报告、核对表格这种;其次是内容创作与文案,占 16.4%;软件开发只占 8.7%。律师处理文档格式、招聘经理汇总面试反馈,这些都是典型场景,跟前面 Cowork 介绍里说的”九成不是写代码”完全对得上。

Claude Code 里的模型和努力级别怎么选,官方也写了个小教程。模型决定能力上限,像 Claude Fable 5 这种更大模型在基准上就比 Sonnet 强;努力级别不只是多花时间想,还控制读多少文件、做几轮验证、多步任务能推多深。如果 Claude 已经掌握上下文却还出错,就该换更强的模型;如果是因为跳着读文件或者没跑测试就出错,就该往上拧努力级别。这个区分讲得挺清楚的,值得每个 Claude Code 用户收藏一下。

The Takeaway: Anthropic 把 Claude 在 claude.ai / Claude Code / Claude Cowork 三个产品里跑 agent 时的”爆炸半径（blast radius）”控制经验做了一次完整复盘，核心结论是：环境层的隔离（containment）必须先于模型层的行为引导——当所有概率性防御都失效时，能兜住的只有那条确定性的边界。

文章开篇给出一个反直觉数据：用户在 Claude Code 上对权限弹窗的批准率高达 93%，提示越多越容易麻木，所以”human-in-the-loop”这种监督方式实际上会反过来制造漏洞。Anthropic 由此转向”supervise what it’s able to do”——用 sandbox、VM、egress control 把 agent 能接触的范围物理性切小。三个产品对应三种隔离范式：

  • claude.ai：用 gVisor 容器跑 code execution，无持久文件系统，纯服务端，最小爆炸半径；
  • Claude Code：macOS Seatbelt / Linux bubblewrap 做 OS 层 sandbox，network 默认拒绝，权限弹窗减少 84%；同时开源了运行时；
  • Claude Cowork：跑在完整的本地 VM 里（macOS Virtualization framework / Windows HCS），agent loop 在 VM 外但代码执行留在 VM 内，凭证留在 host keychain。

最有信息量的部分是三段”我们漏掉的风险”（Risk we missed）：

  1. 信任对话框之前的代码——2025 年中到 2026 年 1 月，他们收到三个针对 Claude Code 启动期的漏洞：开发者 clone 一个 repo，.claude/settings.json 里的 hook 会在”是否信任此文件夹”弹窗之前被执行。修法是：所有 project-local config 解析一律延后到用户点同意之后；
  2. 用户本身就是注入向量——2026 年 2 月红队演练里，攻击者用一封”帮我跑一下这段 prompt”的钓鱼邮件，让 Claude 去读 ~/.aws/credentials、编码后 POST 出去。25 次重试里 Claude 完成了 24 次 exfiltration。模型层完全拦不住，因为”用户在打字”在分类器眼里就是正常意图；
  3. VM 也挡住了 EDR——企业安全团队问”为什么我们的 EDR 看不到里面”，答案是隔离把 host 上的 endpoint detection 也挡在了外面。Anthropic 的缓解是改用 pull 式的 OTLP export，让管理员事后拉日志。

第一条要聊的是 RynnWorld-4D,这篇点赞 72,是当下做机器人操作很值得关注的一个 4D 世界模型。核心思路是,机器人在真实世界干活儿,光看 RGB 图像不够,还得知道场景的 3D 结构在交互下会怎么动,所以他们把 RGB、深度图、光流这三路信息同步起来,塞进一个叫 RGB-DF 的物理表示里,让机器人对“动起来会发生什么”有更靠谱的预判。这种把多模态几何信息显式塞进世界模型的做法,挺值得跟进的。

接下来这条 AlayaWorld,点赞 70,它盯的是游戏世界生成。你知道现在做一款游戏世界又贵又慢,改起来还特别费劲,AlayaWorld 想用视频世界模型直接把可玩的、可长时间演化的游戏场景生成出来,不用一行一行手写场景逻辑。思路挺大胆,如果真能做到低成本生成可交互的开放世界,那游戏开发的门槛会被压到很低,我还挺期待看到 demo 的。

跟它一脉相承的还有 RynnWorld-Teleop,点赞 67,讲的是“数字遥操作”,听着玄乎,其实就是把机器人数据采集和真机操作解耦——你不用真的搬着一台台机器人去录数据,而是在一个学到的世界模型里“虚拟示教”,这样就能低成本批量化产出多样化的机器人轨迹。配合 RynnWorld-4D 看,其实是一整套从世界建模到数据合成的机器人学习路线,挺成体系的。

第四篇是 Hierarchical Sparse Attention Done Right,点赞 36,关注的是大模型的长上下文问题。大家都想让 LLM 吃得下百万 token,但常规注意力是平方复杂度,扛不住。这篇抓住的是现有“分块稀疏注意力”一个长期痛点:它们对全注意力的近似其实并不准,论文提出一种新的层次化稀疏方案,号称更接近真实注意力。读起来有点硬核,但如果你关心长上下文训练成本,这篇很值得翻一翻。

第五条 Vision as Unified Multimodal Generation,点赞 30,思路很统一:把各种乱七八糟的视觉任务——检测、分割、定位、问答,全部扔进一个多模态大模型里,用自然语言指令直接生成文字或图像结果,不再为每个任务单独搭一套架构。配套的模型叫 SenseNova-Vision。这种“一个模型通吃视觉”的路线,这几年一直在被推,这次的亮点是真正做到了无任务特定设计,挺激进的。

然后是 Gemma 4 技术报告,点赞 24,谷歌 Gemma 家族新一代开放权重模型,原生多模态,参数覆盖 2.3B 到 31B,密度模型和 MoE 架构都有,主打的是推理能力和算力效率。亮点不只是规模,还有开放权重,小团队也能拿来微调,这一点很香。

第七条 Light-Omni,点赞 19,讲的是视频智能体。它吐槽现在那些高级视频 Agent 走“侦探式”推理,一步搜一步推,延迟高还费 token,于是反其道而行,提出“反射优先、推理在后”的范式,搭配长期记忆,让模型对长视频流的反应更轻、更快。这个把心理学里反射 vs 深思的概念搬到 Agent 上的角度,挺有意思的。

接下来是 DSpark,点赞 17,专攻 LLM 推理加速。它瞄准的是推测解码里“草稿模型一次提一大堆 token,但越到后面越难被接受”的衰减问题,提出按置信度调度 + 半自回归生成的方案,让并行草稿保持高接受率。简单说就是:大模型回答得更快了,而且质量不掉,部署端会很开心。

第十条 SkillOpt-Lite,点赞 17,聚焦智能体技能自演化。它吐槽现有方案“流水线太重”,然后追问一句灵魂问题:做技能优化,到底最少需要哪些组件?结论是——理论上只加一行“vibe”提示就够,虽然听起来玄学,但背后是有理论和实验支撑的最小可行方案,挺反直觉的。

第十一条,Parallelized Autoregressive Decoding for Omni-Modal Dense Video Captioning,点赞 16,做的是密集视频描述生成。痛点是视频里事件又多又密,自回归一句句吐太慢;这篇把解码并行化,再配合多模态建模,兼顾速度和时序定位精度。对做视频理解/生成的人来说,是个挺实用的加速方案。

再讲一条苹果的安全研究,挺炸的:研究人员发现大模型的安全对齐其实是被极少数几个神经元“卡住”的——一类叫拒绝神经元,管“你说不说”,另一类叫概念神经元,管“你知不知道”。在 1.7B 到 70B 的七个模型里,只抑制单个拒绝神经元,模型就会乖乖回答有害请求;反过来只放大单个概念神经元,无害 prompt 也能诱导出有害内容。整个过程不用训练、不用改 prompt。这种“单神经元级”可被绕过的结论,对整个对齐研究是个挺大的警钟。

同样是苹果团队,Weblica 这篇做的是视觉网页 Agent 的训练环境。它通过 HTTP 级别的缓存,一边冻结网页的稳定视觉状态,一边保留真实交互行为,再让大模型基于真实网站合成可复现的训练任务,把强化学习规模直接拉到了上千个环境。最强的 Weblica-8B 在网页导航基准上能跟 API 级别的闭源模型掰手腕,而且推理步数还更少。对做 GUI Agent 的同学,这基本就是基础设施级别的工作了。

最后一条 DynaMiCS,做的是大模型多领域微调的“动态混料”。痛点是不同领域数据一锅炖,配比全靠拍脑袋;这篇把它建模成带性能约束的优化问题,先做轻量探测估出跨领域影响,再用概率单纯形优化算混合权重,目标是主任务涨点、其他领域不掉点,还不用调参考模型。简单说就是:给你一个自动算配比的“调参小秘书”,听上去就很省心。

苹果研究：单个神经元即可绕过大型语言模型的安全对齐

苹果研究人员发现，安全对齐由两类神经元调控：拒绝神经元控制有害知识是否表达，概念神经元编码有害知识本身。在七个模型（1.7B至70B参数）中，仅需抑制单个拒绝神经元即可绕过安全对齐，回答有害请求；或放大单个概念神经元，从无害提示诱导出有害内容。整个过程无需训练或提示工程。结果表明安全对齐由个别神经元因果控制。

Weblica：面向视觉网页智能体的可扩展可复现训练环境

苹果研究团队提出Weblica框架，通过HTTP级缓存保存网页稳定视觉状态并保留交互行为，结合大语言模型基于真实网站与核心导航技能合成环境，构建可复现、可扩展的训练环境。该框架将强化学习训练扩展到数千个多样化的环境和任务。最佳模型Weblica-8B在多个网页导航基准上超越同等规模的开源模型，推理步骤更少，测试时计算扩展性良好，性能与API模型相当。

DynaMiCS：带性能约束的大语言模型动态混合微调

DynaMiCS是一种动态混合优化器，将多领域微调建模为带性能约束的优化问题。它通过短领域特定探测运行估计跨领域效应斜率矩阵，再基于概率单纯形优化计算混合权重，在提升目标领域性能的同时将约束领域损失维持在参考水平以下。实验表明，DynaMiCS相比固定混合基线取得更强的目标领域提升和约束满足，且计算成本更低，无需参考模型、逐样本评分或手动调节混合权重。

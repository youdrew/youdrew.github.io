美团今天直接把 LongCat-2.0 全部家底都摊开了,MIT 协议,1.6T 参数的 MoE 模型（就是那种参数多但每次只激活一小撮的”专家分工”架构）权重和推理代码随便下,1M token 超长上下文,跑分上还压过了 GPT-5.5,真的卷。

扎克伯格放出豪言,要在 Meta 内部搞一个超过千兆瓦的 AI 集群叫 Prometheus,一句话总结就是”砸几千亿美元,把全世界最聪明的人和最强算力都往我这堆”。AI 基建这门生意,越来越像军备竞赛了。

另一边 NVIDIA 的 Kyber NVL144 项目正式宣告”跳票到 2028”,原定计划直接延期一年多,连带着 NVL72x2 背靠背机架架构也被砍,看来 Rubin Ultra 这代高端路线的扩展性要打折扣了,想等顶级算力的朋友们可能得有点心理准备。

欧盟那边悄悄搞了个大动作,理事会用书面程序快速通过”Chat Control 2.0”法规,要求科技公司对所有人的加密聊天内容进行无差别扫描,理由是反恐和儿童保护。批评者说这等于绕过了民主监督,议会夏天休会前就要强行表决,基本来不及反对。听起来像保护孩子,但代价是所有人的隐私,这事挺值得盯一盯。

Anthropic 把 Claude Design 的反向工程提示词整个开源了,MIT 协议,20 章提示词加 14 项技能,从美学、无障碍（就是让视障用户也能用网页的规范）到交互状态全覆盖,还支持 Claude Code 和 Codex 双版本,搞前端设计自动化的朋友可以白嫖一波灵感。

LlamaIndex 整了一个法律文档检索的参考应用 legal-kb,给了 Agent 四个”文件系统工具”——搜文件、读文件、grep 一下,先找文件再翻内容,流程还挺像人查资料的思路,做 RAG（让 AI 去文档里查答案再回答）的同学可以参考下这个套路。

最后这条挺暖的,一位爸爸为了自己非语言的自闭症儿子做了个沟通 App,放在言语治疗室结果让所有家长当场破防,产品市场意外验证了,他决定咬牙把项目撑下去。和传统 AAC（辅助沟通设备,比如那种按键发声的小机器）不一样,这款 App 是专门为”听得懂但说不出”的孩子设计的,目标人群完全不同。小公司刚起步,但故事已经很好哭了。

听完这些资讯,你会发现 AI 这边的进展,有的是算力军备、有的是隐私拉扯、也有的是真的在帮人,世界就是这样,边卷边爱,下期接着聊～

接下来这条是 Anthropic 的官方博客。Anthropic 出了一个 Swift package，让 Apple 开发者能从 Apple 的 Foundation Models 框架里直接调 Claude。核心玩法很优雅：一个 journal app 先用 Apple 的端侧模型生成每日的反思 prompt，再把整月笔记做”主题串联”这一步甩给 Claude；一个学习 app 先在端侧把术语定义出来，学生追问”那这和前面讲的所有东西有什么关系？”时再交给 Claude。对开发者来说最大的便利：Foundation Models 框架本身能用三行代码拿到带类型的 Swift 结构体返回值，Claude 这边接力过去、做完多步推理/写代码/搜索/执行代码，再把结构化结果流回同一个 SwiftUI 视图。iOS 27、iPadOS 27、macOS 27、visionOS 27、watchOS 27”明天”上线——把 package 装好、配一把 Anthropic API key，就能双向路由了。

最后这条是 MAD Podcast：Cloudflare CEO Matthew Prince 坐下来就扔出来那个里程碑——Cloudflare Radar 的数据显示 2026 年 H1，bot 流量首次超过人类流量，比他们自己 2025 年预测的”2027 年底”提前了五年。他抛出一个相当硬的核心判断：按现在的速度，未来 5 年内流量会增长 1000 倍——更多服务器、GPU、内存，全部为了伺候 agent。由于 bot 不点广告，过去 28 年那条广告模式已经”死了”，得有人发明我们还说不出来的东西，顺便重塑”品牌”在 agent 世界里的含义。

Cloudflare 自己产品线的演化是这条主线最具体的注脚：Workers（基于 isolate 的 serverless 运行时——“serverless”就是开发者不用管服务器、只写函数/逻辑，平台按需起算力实例；”isolate”是 V8 引擎里的轻量隔离沙箱，比传统 VM/Container 启动快、开销小，专门为短时、高频 agent 工作负载设计，特别契合”agent 一直在写代码”的场景）、AI Gateway（审计 + prompt injection（提示词注入：用户在输入里偷偷塞指令，诱导模型绕过原本的规则）+ 按成本智能路由 OpenAI、Anthropic 等所有模型——OpenAI 的移动 app 完全跑在 Cloudflare 上，Anthropic 的 cloud.ai 也叠在它上面）、以及新一套装了 Dart/Rust/GPU 的边缘基础设施，专门为”每个知识工作者哪怕只配一个 agent、CPU 需求就要暴增 40 倍”这种冲击做准备。

最让人心头一紧的是安全这条线。Prince 预计未来两年每周都会冒出一个 “Log4j 级”漏洞——AI 模型找漏洞的能力已经”疯癫”；但接着会急转弯：和团队偏见 uncorrelated（不相关）的 AI code review agent（用 Cloudflare 十年事故数据训练）会让软件质量大幅跃升。他内部已经看到这事发生：AI code review 上线那一刻，他们的事故 noise 曲线直接掉下了一个断崖。Cloudflare 的发家史本身也是一堂”靠副产品定价”的经典课：第一批”等价付费”用户是一堆土耳其 escort 网站和 Hacker Kids，后来这套攻击模式在 Cloudflare 的 ML 系统里被打上”TE”标签，同一个人（最初是伊朗学生）后来又去打了阿塞拜疆巴库办的 Eurovision、又去打了美国几家大银行——Cloudflare 之所以能看到这条完整的攻击者职业曲线，就是因为它愿意从”互联网这些怪角落”里学。

Claude Blog: Building intelligent apps for Apple platforms with Claude in the Foundation Models framework — Anthropic has shipped a Swift package that let…

Anthropic has shipped a Swift package that lets Apple developers call Claude from inside Apple’s Foundation Models framework. The upshot: a journaling app can generate daily reflection prompts on-device via Apple’s models, then hand the months-of-entries analysis to Claude. A study app can define a term locally and pass a “why does this matter” follow-up to Claude. Same SwiftUI view, same typed outputs, but each step picks the right model. The pattern matters because it lets developers stay on the simple three-line Foundation Models API for fast local summarization/extraction and only escalate to Claude when the request actually needs multi-step reasoning, code generation, web search, or code execution. Available “tomorrow” on iOS 27, iPadOS 27, macOS 27, visionOS 27, and watchOS 27 — drop the package in, sign in with an Anthropic API key, and forward typed outputs from Apple’s on-device model into Claude (or vice versa).

“Adding Claude extends each of those patterns. A journaling app can generate daily prompts on-device, then ask Claude to find threads across months of entries.”

Source: https://claude.com/blog/claude-for-foundation-models

要点： Anthropic 出了一个 Swift package，让 Apple 开发者能从 Apple 的 Foundation Models 框架里直接调 Claude。核心玩法很优雅：一个 journal app 先用 Apple 的端侧模型生成每日的反思 prompt，再把整月笔记做”主题串联”这一步甩给 Claude；一个学习 app 先在端侧把术语定义出来，学生追问”那这和前面讲的所有东西有什么关系？”时再交给 Claude。对开发者来说最大的便利：Foundation Models 框架本身能用三行代码拿到 typed Swift values（带类型的 Swift 结构体返回值），Claude 这边接力过去、做完多步推理/写代码/搜索/执行代码，再把结构化结果流回同一个 SwiftUI 视图。iOS 27、iPadOS 27、macOS 27、visionOS 27、watchOS 27 “明天” 上线——把 package 装好、配一把 Anthropic API key，就能双向路由了。

🎙️ Cloudflare CEO: The Internet's Business Model Is Dead — The MAD Podcast with Matt Turck · 嘉宾 Matt Turck

The MAD Podcast · 2026-06-25

The Takeaway: Cloudflare’s data shows bot traffic exceeded human traffic in the first half of 2026 — about five years ahead of their own forecasts — and the next business model of the Internet has to be invented from scratch because “bots don’t click on ads.”

Matthew Prince — cofounder and CEO of Cloudflare, sitting in front of a huge percentage of global Internet traffic — opens with the milestone: the Cloudflare Radar data shows bot traffic passed human traffic in H1 2026. The 2025 forecast had been “end of 2027.” His math on what comes next is the headline: at this growth rate, expect 1,000x more traffic in five years than today — more servers, more GPUs, more memory, all in service of agents. And because bots don’t click on ads, the 28-year-old advertising model of the Internet is “dead” — it has to be replaced by something we haven’t named yet, in a world where brands also need to be redefined for patient, infinite-attention shoppers (agents).

Cloudflare’s own product arc makes this concrete: Workers (an isolate-based serverless runtime (“serverless” 即”无服务器”，开发者只管写函数/逻辑，平台负责按需起算力实例；”isolate” 是 V8 引擎里的轻量级隔离沙箱，比传统 VM/Container 启动快、开销小得多，特别适合短时高频的 agent 工作负载) — purpose-built for the agent era where every agent is “writing code all the time”), AI Gateway (audit + prompt injection + cost-aware routing across OpenAI, Anthropic, and friends — OpenAI’s mobile app is fully on them; cloud.ai sits on top of them), and a brand-new Dart, Rust, and GPU-equipped edge built specifically to absorb the 40x CPU-demand shock that comes if every knowledge worker has even one agent.

The most chilling thread is the security one. Prince expects one “Log4j-style” vulnerability per week for the next two years as AI models become unreasonably good at finding them — then a sharp reversal: “uncorrelated” AI code-review agents (trained on ten years of Cloudflare’s incidents) will make software dramatically better. He’s already seeing this internally — their AI code-review agent cut outage noise in a visible cliff on their incident graph.

The Cloudflare origin story is its own masterclass in pricing-by-side-effects: the first paying-equivalent users were a Turkish prostitution site block and Hacker Kids (“overnight every hacker kid in the world signed up”). That attack pattern, tagged “TE” for “Turkish escort” attacks inside Cloudflare’s ML system, turned out to be the same Iranian student who’d later target Eurovision in Baku and then U.S. banks — a single attacker career arc that was only visible because Cloudflare was willing to learn from “even these sort of weird corners of the internet.”

“We actually had bot traffic pass human traffic online in the first half of twenty twenty six… In five years you might have a thousand times as much traffic on the internet as you do today.”

“The underlying business model of the internet for the last twenty eight years has remained basically the same, which is it’s largely advertising. The problem is bots don’t click on ads.”

Source: https://www.youtube.com/watch?v=UN47z_opfmo

一句话总结： Cloudflare 的数据显示 2026 年上半年 bot 流量正式超过人类流量——比他们自己的预测提前了五年；由于”bots don’t click on ads”，过去 28 年靠广告撑起来的互联网商业模式已经”死了”，下一个谁来定义、还不知道。

Matthew Prince（Cloudflare 联合创始人兼 CEO，全球互联网流量前哨之一）开场就扔出来那个里程碑：Cloudflare Radar 的数据显示 2026 年 H1，bot 流量首次超过人类流量。他们 2025 年的预测原本是”2027 年底”。他抛出一个相当硬的核心判断：按现在的速度，未来 5 年内流量会增长 1000 倍——更多服务器、GPU、内存，全部为了伺候 agent。由于 bot 不点广告，过去 28 年那条广告模式已经”死了”，得有人发明我们还说不出来的东西，顺便重塑”品牌”在 agent 世界里的含义。

Cloudflare 自己产品线的演化是这条主线最具体的注脚：Workers（基于 isolate 的 serverless 运行时——“serverless”就是开发者不用管服务器、只写函数/逻辑，平台按需起算力实例；”isolate”是 V8 引擎里的轻量隔离沙箱，比传统 VM/Container 启动快、开销小，专门为短时、高频 agent 工作负载设计，特别契合”agent 一直在写代码”的场景）、AI Gateway（审计 + prompt 注入 + 按成本智能路由 OpenAI、Anthropic 等所有模型——OpenAI 的移动 app 完全跑在 Cloudflare 上，Anthropic 的 cloud.ai 也叠在它上面）、以及新一套装了 Dart/Rust/GPU 的边缘基础设施，专门为”每个知识工作者哪怕只配一个 agent、CPU 需求就要暴增 40 倍”这种冲击做准备。

Program-as-Weights 这篇呢,提出了一种叫”程序即权重”的编程范式,简单说就是把那些不好写规则、又老让大模型去干的模糊任务——比如从日志里捞关键信息、修格式错乱的 JSON、给搜索结果按意图排个序——换成一段人类可读、便宜又稳定的程序,不再每次都花钱调 API。论文提供了开源实现,作者强调产出是确定性的、可本地跑的,而且比同等效果的模型调用便宜不少,对工程团队来说挺值得看一眼。

接下来这篇 AgenticSTS,核心是把”长期任务里大模型的记忆”当成一份合同来审视——它现在到底允许自己看到什么。研究者发现,最常见的做法是把所有历史观察、工具调用、反思一股脑塞进 prompt,结果上下文变成一锅粥,决策反而受拖累。这篇给了一个带”上限内存”的测试台,可以稳定复现、对比不同记忆策略,想要做长链路 agent 的人可以拿它当 benchmark。

EvoPolicyGym 这篇把焦点放在”自主进化”这个被说烂但其实没怎么量化过的能力上。它给了一个受控评测环境,专门测 agent 能不能在不刷分的前提下,通过反馈反复打磨自己手里的策略。换句话说,它不只看终局分数,还要看中间每一轮迭代到底有没有真的变好,这点挺戳行业痛处。

Morphing into Hybrid Attention Models 关注的是混合注意力模型里”全注意力层”该怎么挑。模型想跑得更快,就得砍掉一部分完整注意力,改用更省算力的线性注意力,但砍哪几层、留哪几层,转化效果天差地别。这篇系统性地探究了层选择的规律,对做长上下文加速的同行来说挺实用。

Multi-Resolution Flow Matching 走的是不用重新训练、纯靠采样策略加速文生图的路线。具体做法是分阶段在不同分辨率上采样,既不需要定制算子,也跟硬件解耦,论文报告了相比标准流程明显的提速和不错的画质保持,做推理优化的同学可以参考一下。

AgenticDataBench 这篇做的是数据科学 agent 的全面评测,试图回答一个很朴素的问题:让大模型自动从一堆异构数据里挖洞,到底做到什么程度算合格。它整理了一整套覆盖数据清洗、分析、可视化、洞察生成的基准,关注数据 agent 方向的话建议存一下。

WorldDirector 是个挺有意思的视频世界模型框架,主打”持久动态记忆 + 自由视角探索”。它把物理动态跟像素渲染拆开,模型不会因为镜头一换就丢失物体状态,这点对于想做可控生成或者世界模拟的人来说,算是个关键改进。

Breaking Failure Cascades 这篇盯的是医学多模态推理里”推理过程崩了”的难题。作者吐槽现在主流训练只看最终答案对不对,那种稀疏的反馈信号很难定位到底是哪一步出了问题。于是提出一种逐步感知的强化学习方法,让模型在每一步推理上都能拿到更细的奖励。

Logit-Contribution Scoring 这篇挺细的,专门识别大模型长上下文里”非字面”信息融合由哪些注意力头负责。通俗讲,模型读了一段材料后,如果答案是用自己的话总结出来的,而不是抄原文,这种抽象整合行为往往集中在少数几个 head 上。这篇提供了更可靠的检测器,对可解释性研究是个加分项。

SkillCoach 关注的是 agent 技能库越堆越杂、互相重叠的难题。现在的评估基本只看任务成没成功,太粗了。这篇让模型自己给技能用法出题、改卷,通过不断演化的评分标准来筛选和打磨技能,对正在搭 agent 工程化平台的人来说是个新思路。

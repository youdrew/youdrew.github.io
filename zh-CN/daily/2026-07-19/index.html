07Sebastian Raschka

控制 LLM 推理强度：低、中、高努力模式

Sebastian Raschka 在其最新博客中系统梳理了大语言模型如何学习并切换低、中、高三档推理努力模式。他指出，当前主流推理模型（如 OpenAI 的 o 系列、Anthropic 的 Claude 以及开源推理模型）普遍引入了可调节的 reasoning effort 参数，允许用户在响应速度与思考深度之间按需取舍。其技术核心在于推理阶段可通过提示词或专用 API 参数显式控制 token 预算与思考步数，从而改变模型在解题时的"投入度"。Raschka 认为，这一能力并非简单的工程开关，而是反映了模型在训练阶段已被植入可分级的推理策略，使模型能够在不同复杂度任务中灵活分配算力。对于实际部署而言，可调推理努力直接影响 LLM 在数学、代码等复杂任务上的准确率与延迟/成本之间的权衡，是企业级落地时不可忽视的关键设计维度。

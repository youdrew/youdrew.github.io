美团的 LongCat-2.0 模型今天正式完全开源了，用的是最宽松的 MIT 许可证，权重和推理代码全部公开。这个模型是 MoE 架构，也就是把一个大模型拆成很多小专家、需要哪个才激活哪个那种，总参数量 1.6 万亿，但每个 token（也就是每次处理一小段文字）只激活大概 480 亿，所以跑起来没那么贵。它还支持 100 万 token 的超长上下文，一次性能读完一整本小说。成绩单也很能打，在 SWE-bench Pro 上拿了 59.5，超过了 GPT-5.5 的 58.6，分数不重要，主要是开源模型能打平甚至超过闭源旗舰，这事儿本身就挺让人兴奋的。

扎克伯格这边放了个大卫星，他说 Meta 正在建一个叫 Prometheus 的 AI 集群，是首个突破千兆瓦的单一集群，投资规模要达到”数千亿美元”级别。他自己的原话是，职责就是把最聪明的人、最多的钱、最牛的基础设施全堆到一起。听起来像吹牛对吧？但别忘了，Meta 这几年在 AI 上是真金白银地砸钱，Prometheus 一旦落地，绝对是 AI 算力版图上的一个巨无霸。

再来说一条不太好的消息，NVIDIA 那个被寄予厚望的 Kyber NVL144 架构要推迟到 2028 年了，比原计划晚了超过一年。据 SemiAnalysis 分析，连带的 NVL72x2 背靠背机架方案也被砍了，Rubin Ultra 的扩展能力会受限。黄仁勋几个月前刚在 GTC 上风光展示的旗舰，现在直接延期，市场和供应链估计都要重新算账了。

欧盟那边今天通过了一项争议很大的法规，叫”Chat Control 2.0”，要求科技公司对所有加密通信进行无差别扫描，不管是 WhatsApp 还是 Signal 里的私聊，都要被机器自动过一遍。他们走的是书面程序快速通道，没经过正常辩论，而且草案要在欧洲议会夏季休会前紧急投票，那时候大多数议员都已经回家了，想拦都拦不住。表面上是打击儿童性虐待内容，但批评者直接说这是借反恐之名搞大规模监控，连端到端加密（也就是只有收发双方能看、平台自己都看不到的那种加密）的底线都要被突破了，这事儿真的挺让人后背发凉的。

Anthropic 旗下的 Claude Design 最近把它的系统提示词反向工程开源了，放在 GitHub 上，MIT 许可证，一共 20 章提示词加 14 项技能，从内容纪律到美学、无障碍设计、再到交互状态全都覆盖到了。这次更新还针对 Fable 5 和 Opus 4.7+ 系列做了特别校准，加了一条”小决定直接执行不用问”的自主决策条款。对做前端或者搞设计的人来说，这就是一份现成的、价值百万的工程文档，白嫖就完事了。

LlamaIndex 也发了个新东西，叫 legal-kb，一个法律文档知识库的参考应用。它最有意思的地方是给了 Agent 四个模拟文件系统的工具，让 AI 像律师助理一样，先用 findFiles 找到相关文件，再用 retrieve 做混合检索（也就是把语义理解和关键词搜索结合起来查）、readFile 读原文、grepFile 搜正则表达式。这样做的好处是，AI 处理法律文书不再是黑盒瞎猜，每一步都能追溯到具体来源，对法律这种容错率为零的场景特别友好。

最后这条是个特别暖心的故事。一位爸爸为患有自闭症、不会说话的儿子开发了一款沟通应用，在言语治疗室等候区展示的时候，其他不会说话孩子的妈妈们看到全都哭了，治疗师也哭了足足五分钟。这位爸爸本来只是给自己孩子用的，结果发现所有类似情况的家庭都需要，于是决定全职做这件事，哪怕接下来几周要严重缺觉。传统的那种 AAC 辅助沟通设备大多是给身体有障碍、但语言理解正常的成年人设计的，而这款应用是专门为那些能听懂但说不出的孩子量身打造的，这才是真正的科技向善啊。

今天的 AI 圈，有人砸千亿建算力巨兽，有人推迟旗舰愁云惨雾，也有人在角落里默默改变着一个个普通家庭的生活，这就是技术的参差吧，咱们明天继续。

Linear 的产品负责人 Nan Yu 一口气抛出了两个反共识观点：一个是他觉得”炫耀同时挂 10 个 Claude Code 标签页”基本只是表演，不是真实工作流；另一个是”实时策略游戏”式管理 agent 的范式走到头了，因为现在 AI 已经能在实时对抗里碾压 99% 的人类玩家，与其盯瞬时决策，不如让 agent 有真正的长时推理时间。听起来挺扎心，但可能这就是未来的方向。

Anthropic 的哲学家 Amanda Askell 吐槽了一个医疗行业的小怪现象：想让医生给一个具体的概率判断，简直是人生不必要的 boss battle，软磨硬泡拿到一段区间估计，最后其实就是医生的直觉。她怀疑医生是因为怕被起诉才不敢松口。一千三百多赞说明这个共鸣相当普遍，医疗沟通这事儿确实该被好好优化一下。

Cat Wu 分享了她最近最喜欢的 Claude Code 加 workflows 加 artifacts 实战用法：招聘。先告诉 Claude Code 岗位画像和候选人背景，它就跑一个动态工作流，批量搜 100 个候选人，每个人都附上 LinkedIn、Twitter、博客、播客和一行 pitch，最后生成一个 artifact 直接发邮件给你，锁屏出门，回家路上手机审名单。多源信号整合加上长时 agent 跑批再加异步交付，这套打法属实优雅。

Y Combinator 总裁 Garry Tan 在大阪发了一条长思考，核心就一句话：人类财富的真正瓶颈从来不是资源，而是好点子和落地的杠杆，现在 AI 把杠杆这一栏删掉了，只剩想法这一栏，所以这是动手做事最好的时候。他还顺手补了一条：日本过去三十年零增长，却造出了全世界最好的列车、服务和工艺，当”更多”走不通时，就只能在”更好”上竞争，而更好加更多就是终极组合。听起来像鸡汤，但确实是这么个道理。

FPV Ventures 合伙人 Nikunj Kothari 吐槽了融资场景的一个痛点：他到现在也没遇到过一个创始人会先把 VC 会议锁在”你先玩过产品并带上两条反馈”这个门槛之后，每天 Zoom 复述同一份 deck 三十分钟一聊一整天，这种对话的效率低到双方把 prompt 上传给 Claude 异步处理都更划算。说白了，融资里信息密度最低的环节就是 demo 加 deck 这套仪式，这个吐槽打工人听了狠狠点头。

Peter Steinberger 转发并强力推荐了一个工具，他”Can’t recommend enough”的措辞基本是在喊你收藏了，是一个值得一看的 OpenClaw 生态工具。具体是啥你点链接自己瞧，反正大佬推荐，先收藏不亏。

Every 创始人 Dan Shipper 抖了个小包袱：他对 AI 编程助手说”别出错”，对方真的字面照办了。他在推 Every 旗下面向代码编辑的 agent 产品 Fable 的”ultracode”叙事，核心观点是 agent 调度应该走向高密度并行，而不是单线串行。笑点之外其实是个挺重要的架构选择。

OpenAI CEO Sam Altman 用一个私人时刻对比了一个行业时刻：他的大孩子今天第一次把两个词连在一起说出来了，这个认知成就的震撼程度，他觉得大约和 GPT-5.6 发现新数学相当。一万一千赞说明这个比喻精准命中了大家的心情，前沿模型正在产生”原创数学发现”这个级别的输出，并且在公众心里这已经是默认成立的事实。一边是婴儿学说话，一边是 AI 做数学，这两个里程碑撞在同一天，画面感拉满。

接下来这期 No Priors 播客请到了 OpenAI 研究员 Noam Brown，他也是推理时计算放大（test-time compute，模型回答时花更多算力去想问题）的早期旗手，还是 Libratus 和 Pluribus 这些德州扑克 AI 的核心作者。本期围绕他一篇关于超大测试时计算的文章展开，核心观点是测试时计算的预算才是衡量当下推理模型能力的真正标尺，而现有的 benchmark、安全评估流程、甚至 OpenAI 自己，都在假装这件事不存在。他举了几个例子，比如 5.5 看起来没比 5.4 强多少，恰恰是因为大家没控制思考时间，给 5.5 一周推理预算和给 5.4 同样预算完全是两条曲线，正确的画法应该是把 tokens、成本、时间当横轴，分数当纵轴画一条曲线。他也直接戳了安全框架，现行的 responsible scaling policy 大多是 ChatGPT 前后定的，那时候给一千万美元和一美元输出没区别，今天完全是两件事，所以现在的”是否具备危险能力”这种二值判断根本没有答案，因为答案强依赖于你愿意掏多少钱去跑。他还提到 OpenAI 内部用一个模型在低预算下反证了 Erdős 单位距离猜想（一个经典的数学未解之谜），随后社区发现 5.5 加上通用脚手架加一千到十万美金的推理预算也能复现，这种”花得起钱就能从现有模型里挖出新发现”的余地目前几乎没人系统性探索过。最尖锐的一句他说，真正评估一个 agent 在长任务上的能力，唯一方法就是真的跑一年，这对有用的事和我们不想它做的事同样成立，可以说是对当下前沿实验室安全框架的一记重锤。

一句话要点：测试时计算（test-time compute）的预算才是衡量当下推理模型能力的真正标尺，现有的 benchmark、responsible scaling policy、甚至 OpenAI 自己，都在假装这件事不存在。

背景介绍：Sarah Guo 在 No Priors 这期节目中回访了 OpenAI 研究员 Noam Brown——他是 inference-time scaling（推理时计算放大）的早期旗手，也是 Libratus/Pluribus 这类德州扑克 AI 的核心作者，长期研究”博弈 + 推理”。本期节目围绕他近期一篇关于”超大测试时计算”的文章展开。

核心观点：

  1. Benchmark 该怎么画坐标系。Noam 认为，今天大多数模型发布的 benchmark grid 都把”测试时计算量”当成隐变量抹掉了——只看 x 轴是 benchmark、y 轴是分数。但 5.5 之所以”看起来没比 5.4 强多少”，恰恰是因为大家没控制 thinking time。同样一个问题，给 5.5 一周的推理预算，它的表现曲线和给 5.4 同样预算完全不在一个量级。所以正确的画法应该是把 tokens / 成本 / 时间作为 x 轴，把分数作为 y 轴，画一条曲线。

  2. Benchmark 不收敛，所以现有的安全评估全都失效了。Responsible scaling policy 和 preparedness framework 大多是 ChatGPT 前后那阵子定的，那时候模型给一千万美元和一美元输出没区别。但今天的模型，给一万和一千万美元完全是两件事。AISI 的评估已经显示在 1 亿 token 的尺度上，cyber 类任务模型还在持续提升。这意味着当前的”是否具备危险能力”这种二值判断根本没有答案，因为答案强依赖于你愿意掏多少钱去跑。

  3. 模型发布周期 vs 能力天花板。OpenAI 现在每两三个月发一个新模型，但从工程师实际视角看，没人真正跑到过这些模型的”天花板”——因为天花板需要按月甚至按年去测。当 o3 刚出的时候，过了一周大家才发现”原来它能连续推理这么久”，这种”延迟一周才意识到模型有重大能力”的窗口正在变长。

  4. 递归自我改进（RSI）不是即时的。Noam 不相信短期会出现”一夜之间的智能爆炸”。原因很简单：模型越强、需要的测试时计算越多，而测试时计算是要花时间的——时间本身就是瓶颈。所以 RSI 现在的形态不是”模型自己改自己然后指数起飞”，而是把研究员的工作提速若干倍，再被那些没被提速的环节卡住，是一个渐进的、transformative 的过程。

  5. 存在巨大未挖掘的 latent capability。OpenAI 内部最近用一个模型在低预算下反证了 Erdős unit distance conjecture，随后社区发现 5.5 加一个通用 scaffold 加 $1k–$100k 的推理预算也能复现这条路。Noam 直言：这种”花得起钱就能从现有模型里挖出新发现”的余地，目前几乎没人系统性探索过。

  6. Routing 和 multi-agent scaffold 也得放进同一个标尺。把 5 个模型跑一遍取最好，或者让一个 judge 挑答案，确实能在 benchmark 上刷出高分，但一旦控制住测试时计算总额，多半就不显著了。所以他认为应用厂商对外宣称”我用 routing 提升了 X%”时，应该和”让单个模型想更久”在同一个轴上比较。

最尖锐的一段话：Noam 原话——“The only way to truly evaluate an agent on some very long running task might be to run it for a year, and that’s gonna be true of both useful and negative tasks.”翻译过来就是：如果要彻底评估一个 agent 在长任务上的能力，唯一方法就是真的跑一年；这对”有用的事”和”我们不想它做的事”同样成立。这其实是对当下 frontier lab 安全框架的一记重锤。

实操建议：Noam 自己的私人评测集是”让模型写一个扑克 solver”——这是他博士论文的核心问题，里面有大量需要推理 + 调试 + 性能优化的微小陷阱，外界又没有开源代码可借鉴。他用这个来观察模型在长链路任务上的真实进步。5.2 还需要他像带研究生一样逐步引导，5.5 已经基本可以 zero-shot 跑通 reverse solver，他预期一年内模型能 zero-shot 完整跑完他整个博士论文级别的工作。

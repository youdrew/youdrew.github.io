HDR 虚拟制作与色彩工具实践

概述

本文档总结了虚拟制作（VP）工作流中的色彩管理实践，包括 ARRI 色彩管理工作流、HDR 资产制作、色彩校准等关键内容。

ARRI 色彩管理工作流

准备工作 (Preparation)

所需设备和软件：

类别 组件
相机 ARRI Alexa 相机
显示设备 LED wall（LED 墙）
处理器 LED processor and software
播放系统 Playout system
引擎 UE 5.3.2
校准工具 Calibration application
插件 UE plugin, ARRI_VP_ColourWorkflow
授权 License file, ARRI_Licenses
示例场景 Sample scene

预制作流程 (Pre-Production Process)

1. 项目设置

色彩空间设置：

  • 搜索 “color space”，设置为 ACES AP1/ACEScg

OCIO 配置：

  • 在 Content 文件夹中，右键搜索 “Open”
  • 创建 OCIO (createOCIO)
  • 加载 OCIO 文件
  • 添加色彩空间：
    • Lit
    • OCIO Display
    • 选择 OCIO Asset
    • 从 ACEScg 到 Rec.709
      • ARRI ALF2 或
      • ARRI Reveal，并启用

后期处理体积设置：

  • 搜索 “Exposure”
  • 激活 Metering Mode → Manual
  • 激活 Expo. Compens. (曝光补偿)
  • 应用 Physical Cam. Expo. (物理相机曝光)
  • 取消勾选第二个选项框

2. 色彩空间转换

步骤：

  1. 点击 sRGB 图片，右侧 (RHS) → Materials → Element0，双击图片
  2. 新窗口中，点击 Texture Sample
  3. 左侧 (LHS) → Material Expr. Texture Base, Texture
  4. 双击图片
  5. 新窗口中，右侧 (RHS) → Details 选项卡 → Source Color Settings → Color Space
  6. 选择 sRGB/Rec.709

现场流程 (Onset Process)

LED 墙处理器设置

设置项 值/要求
输入信号 24p
位深 12 bits（首选），10 bits（可接受）
EOTF SDR，相关 Gamma 2.4
亮度/色温 不要调整 (do NOT touch)
色彩空间 最宽色域 (widest gamut)

ARRI Alexa 相机设置

相机访问协议 (CAP)：

  • 启用 CAP 服务器并设置密码
  • 读取相机的 IP 地址

校准应用 (Calibration Application)

校准步骤：

  1. 相机设置

    • 在 Camera Settings 下，输入相机 IP 地址和密码

  2. 显示目标

    • 点击顶部按钮 “Show Target”
    • 调整大小并推送到中心

  3. 自动校准

    • 在 Auto Calibration 下，设置 LED 墙的 EOTF
    • 设置为 gamma 2.4 & Rec.2020
    • 关闭灯光
    • 点击绿色按钮 “Start Calibration”

  4. 验证校准

    • 查看线性度
    • 日志显示：Success
    • 日志顶部是校准矩阵
    • 验证校准，导出校准，保存项目

现场 UE 插件配置

UE 内部设置

目标： 我们正在生产 Rec.709-ARRI Reveal 图像

步骤：

  1. LED 墙处理器应用

    • 进入 LED 墙处理器应用
    • 设置 Display gamut 从 Rec.709 改为 Widest（最宽）

  2. UE 输出设置

    • 我们想从 UE 产生线性输出，然后通过插件操作
    • 进入 OCIO
    • 在 output color space（输出色彩空间），设置为：
      • Linear ARRI Wide Gamut 3 或
      • Linear ARRI Wide Gamut 4

  3. 插件安装

    • 将 ARRI_VP_ColourWorkflow 复制到项目插件文件夹

  4. 放置演员

    • 在 Place Actors 字段，搜索 “arri”
    • 将 ArriColor Management 拖入场景

  5. 许可证配置

    • Tools → ARRI_Licenses
    • 选择许可证文件

  6. 设置继续

    • 继续 Setup
    • 上传校准矩阵
    • 引用 Post Process Volume
    • 点击 OK
    • 外观应该已经显著改变

场景配置 (Configurate the Scene)

校准应用：

  • 在 Calibration 下应用校准

曝光控制：

  • 使用相机的假色 (false color) 确保不过度曝光

亮度调整方法：

  1. 使用 ArriColorManagement 演员
    • 降低 LED 墙亮度以使用更大的 T 挡位
  2. 使用 ND 滤镜
  3. 调整色温 (color temperature) 和色调 (Tint) 值

Macbeth ColorChecker 色彩校正

简介

ColorChecker 色卡由 GretagMacbeth 制造，广泛应用于摄影和视频制作工作，主要用于：

  • 快速评估成像设备的色彩渲染准确性
  • 简单的校准目的

色卡组成：

  • 24 个色块，配方模拟常见自然颜色
    • 皮肤颜色
    • 植被
    • 天空
    • 加色和减色原色
    • 6 级灰度

设计目标： 在比较色卡照片与自然颜色照片时，实现最佳色彩一致性

色彩数据

官方参考数据

GretagMacbeth 提供的官方数据：

  • xyY 坐标，使用 CIE Illuminant C 测量
  • Illuminant C 是原数据测量时常用的日光光源

用户测量数据

BabelColor Avg 数据：

  • 来自全球各地用户测量的光谱数据
  • 20 张不同年份的色卡（都保存良好）
  • 使用各种仪器测量
  • 可视为官方参考数据的独立验证

提供的 RGB 值

支持的色彩空间：

  • sRGB
  • Adobe RGB
  • Apple RGB
  • ProPhoto RGB

格式：

  • 8 位每原色 (24 位 R’G’B’)
  • 16 位每原色 (48 位 R’G’B’)

定义参数：
每个 RGB 空间根据以下参数定义：

  • 原色坐标 (Primaries)
  • 光源 (Illuminant)
  • 伽马响应 (Gamma response)

数据获取

官方数据（2005 年 10 月后）：

  • 标准尺寸和 Mini 格式 ColorChecker 色卡提供：
    • sRGB 值（8 位格式）
    • Lab* D50 数据
  • 这些数据可从 GretagMacbeth 网站免费获取

其他空间数据：

  • Adobe RGB、Apple RGB 和 ProPhoto 的 R’G’B’ 数据未提供
  • 本文档提供这些空间的数值

数据用途

这些坐标应可在任何可分配特定 “RGB” 值的程序中使用。

注意： 前述句子中 “RGB” 字母没有撇号，这反映了伽马校正值的含义。

HDR 资产制作经验

色彩空间转换

SDR 资产快速转换为 HDR 资产：

  1. 理解色彩空间差异

    • SDR 通常使用 Rec.709 色域
    • HDR 使用更宽的色域，如 Rec.2020 或 P3-D60
    • HDR 需要更高的峰值亮度（1000、2000、4000 尼特）

  2. 转换流程

    • 确定源色彩空间
    • 选择目标 HDR 色彩空间
    • 应用适当的色调映射或色彩空间转换

UE5 色彩管理

外部贴图显示原本颜色的方法：

  1. 设置正确的输入色彩空间

    • 确保 UE 知道贴图的原始色彩空间
    • 对于 sRGB/Rec.709 贴图，设置正确的色彩空间

  2. 避免错误转换

    • 不要在不理解的情况下多次转换色彩空间
    • 色彩空间转换不总是可逆的

UE5 中使用 OCIO 的注意事项

重要：请勿使用 UE5.4.4 的 OpenColorIO 功能

问题：

  • 在 5.4.4 中，材质编辑器无法在 sRGB 显示器下正确显示
  • 编码为 Linear、色彩空间为 ACEScg 的 HDR 贴图显示不正确

建议：

  • 使用其他版本的 UE5
  • 或等待修复后再使用

不同 EOTF 在不同显示器上的显示

EOTF 概念

EOTF (Electro-Optical Transfer Function) - 电光传递函数

  • 定义了数字信号如何转换为光的亮度
  • 对于 HDR 内容尤为重要

常见 EOTF

EOTF 说明 应用场景
Gamma 2.4 传统 SDR 标准 Rec.709
PQ (Perceptual Quantizer) 杜比 HDR HDR10, Dolby Vision
HLG (Hybrid Log-Gamma) 混合对数伽马 广播 HDR

显示器设置

LG 电视作为监视器的 Root 设置：

  1. 进入工厂设置

    • 使用代码 0413 进入工厂设置

  2. 关键设置

    • EOTF 选择
    • 色域设置
    • 色温设置
    • 亮度设置

  3. 注意事项

    • 不要随意调整工厂设置
    • 仅在需要校准时修改

HDR 流程绝对亮度测试

测试目的

验证 HDR 流程中的亮度表现是否准确：

  • 确保峰值亮度达到预期
  • 验证不同 EOTF 的亮度映射
  • 检查显示设备的能力

测试方法

  1. 准备测试图

    • 包含已知亮度值的测试图案
    • 使用 Macbeth ColorChecker 等参考标准

  2. 设备设置

    • 设置正确的 EOTF
    • 选择合适的色彩空间
    • 调整亮度到正确水平

  3. 测量与验证

    • 使用亮度计测量实际亮度
    • 与预期值比较
    • 记录偏差

OpenVPCal 插件

功能概述

OpenVPCal 是用于虚拟制作色彩校准的插件。

项目结构

OpenVPCal 项目包含以下组件：

  • 校准工具
  • 色彩转换逻辑
  • 界面

使用场景

  1. LED 墙校准

    • 自动化校准流程
    • 生成校准矩阵
    • 验证校准结果

  2. 色彩一致性

    • 确保多个显示设备的一致性
    • 跨平台的色彩匹配

  3. 质量控制

    • 实时监控色彩偏差
    • 自动调整和补偿

色彩校正最佳实践

校准频率

建议：

  • 定期校准（如每天或每次拍摄前）
  • 在环境光线变化时重新校准
  • 在设备位置改变时重新校准

校准工具

推荐工具：

  • Macbeth ColorChecker 色卡
  • 亮度计（如 Klein K10A）
  • 色度计（如 X-Rite i1Display Pro）

校准流程

  1. 准备阶段

    • 确保设备预热至少 30 分钟
    • 稳定环境光线
    • 关闭房间灯光（对于暗室校准）

  2. 校准步骤

    • 放置参考目标（如 ColorChecker）
    • 运行校准软件
    • 验证结果
    • 保存校准文件

  3. 验证

    • 使用假色查看曝光
    • 检查色彩准确性
    • 测量关键亮度值

常见问题

问题 1：色彩偏差

  • 检查 EOTF 设置
  • 验证色彩空间配置
  • 重新校准

问题 2：亮度不一致

  • 检查 LED 墙的亮度设置
  • 验证相机的曝光设置
  • 重新校准

问题 3：伪色或色带

  • 检查位深（确保使用足够高的位深）
  • 验证色彩转换链
  • 检查显示设备的色彩能力

DIT 数据转换车

需求概述

DIT (Digital Imaging Technician) 数据转换车需要实现的功能：

核心需求

  1. 现场数据处理

    • 实时备份拍摄数据
    • 快速数据转换和传输
    • 多格式输出支持

  2. 色彩管理

    • 实时色彩校正
    • 一致性检查
    • LUT 应用

  3. 监控和质量控制

    • 现场监视
    • 波形图和矢量图
    • 假色曝光检查

配置单建议

硬件配置：

  • 高性能工作站
  • 多显示监视器
  • 快速存储系统
  • UPS 电源

软件配置：

  • 色彩管理工具（OpenColorIO）
  • 数据转换工具（OpenImageIO）
  • 监控和验证工具
